Справочник предназначен для примера использования ряда подсистем.

В подсистеме Банки в нем хранятся сведения о банковских счетах всех юридических и физических лиц: собственных и сторонних.

  • При заполнении реквизитов банковского счета необходимо указать владельца счета, наименование счета, номер счета, валюту денежных средств, банк, в котором открыт счет и банк – корреспондент для непрямых расчетов, а также дополнительную информацию, необходимую для корректного заполнения платежных документов по выбранному счету.
  • Для выбора банка достаточно заполнить поле БИК. При вводе значения БИК происходит поиск банка в классификаторе банков РФ и подстановка значений в остальные поля. В случае если банк с таким БИК отсутствует в классификаторе банков можно ввести информацию о реквизитах нового банка, установив флаг Ручное изменение реквизитов банка, или Ручное изменение реквизитов банка для непрямых расчетов.
  • Банковские счета можно вводить в справочнике Организации, в документах Демо: Списание безналичных денежных средств и Демо: Счет на оплату покупателю.

 

В подсистеме Даты запрета изменения элементы справочника используются как "разрез" дат запрета изменения Банк, который используется в документах Демо: Списание безналичных денежных средств.

В подсистеме Управление доступом справочник показывает пример ограничения по одному реквизиту составного типа, а также пример различий в ограничениях для сеансов разного назначения (для пользователей и для внешних пользователей).

  • Для пользователей список ограничивается по одному из 2-х видов доступа в зависимости от типа значения:
    • Организации (проверяется значение реквизита Владелец, если значение владельца - организация);
    • Демо: Группы партнеров (проверяется значение реквизита Владелец.Партнер, если значение владельца - контрагент).
  • Для внешних пользователей список ограничивается по двум видам доступа:
    • Организации (проверяется значение реквизита Владелец, если значение владельца - организация);
    • Внешние пользователи (проверяется значение реквизита Владелец.Партнер, если значение владельца - контрагент).
    • Внешние пользователи (проверяется значение реквизита Ссылка элемента справочника Внешние пользователи, у которого значение реквизита Объект авторизации = значению реквизита Ссылка элемента справочника Демо: Контактные лица, у которого в свою очередь значение реквизита Владелец = значению реквизита Владелец.Партнер элемента этого списка);
    • достаточно, чтобы значение было разрешено в рамках одной проверки по любому из двух видов доступа (в одной из групп доступа).
