Документ предназначен для примера использования ряда подсистем.

В подсистеме Взаимодействия справочник демонстрирует пример контакта взаимодействий.

В подсистеме Контактная информация справочник демонстрирует пример размещения контактной информации объекта-владельца и физического лица. При этом демонстрируется поддержка обратной совместимости ранее созданной таблицы контактной информации, у которой отсутствуют реквизиты, задействованные в новых возможностях подсистемы: ИмяЭлементаДляРазмещения, ХранитьИсториюИзменений, ДействуетС, ЭтоИсторическаяКонтактнаяИнформация.

В подсистеме Пользователи элементы справочника используются в качестве объекта авторизации внешних пользователей, имеющих доступ к информационной базе для работы с задачами, анкетами, счетами на оплату и другими объектами и подсистемами, которые предназначены для внешних пользователей.

В подсистеме Управление доступом показывает пример различий в ограничениях для сеансов разного назначения (для пользователей и для внешних пользователей).

  • Для пользователей список ограничивается по одному виду доступа:
    • Демо: Группы партнеров (проверяется значение реквизита Владелец).
  • Для внешних пользователей список ограничивается по одному виду доступа:
    • Внешние пользователи (проверяется значение реквизита Владелец);
    • Внешние пользователи (проверяется значение реквизита Ссылка элемента справочника Внешние пользователи, у которого значение реквизита Объект авторизации = значению реквизита Владелец этого списка);
    • достаточно, чтобы значение было разрешено в рамках любой из двух проверок по виду доступа (в одной из групп доступа).
